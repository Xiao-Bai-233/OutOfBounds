一、关卡定位

关卡性质

第一关不只是新手教学，而是整个项目的核心玩法验证关。

它要在 8-12 分钟内让玩家和评委确认四件事：

UI 物理化是稳定可玩的
UI 松手后会受重力、碰撞、摩擦影响，不是悬浮摆件。

不同 UI 有不同谜题规则
文本、窗口、血条、对话框、下拉菜单、复选框不是同一种砖块换皮。

双轴操控有真实价值
玩家需要用键盘控制 Caret-7，同时用鼠标介入界面层。

关卡不会被万能拖拽轻易逃课
每类 UI 都有上下文、区域或功能限制，避免“一块 UI 拖到底”。

这个设计也回应了原提案中评委最可能追问的问题：物理 UI 是否已稳定验证、涌现式玩法是否真实、不同 UI 是否能支撑后续关卡，而不是只靠概念描述。


主角状态

Caret-7 从一个损坏输入框中掉落，系统文字闪烁：

Init_Error: Input entity detached.
Unknown cursor object detected.


三、第一关核心规则

规则 1：UI 抓取态 / 释放态 / 稳定态

所有可物理化 UI 都有三个状态。

抓取态

鼠标按住 UI 时，UI 被“外部权限”牵引。
它不会完全瞬移跟手，而是带一点重量感追随鼠标。

释放态

松手后立刻恢复物理。
UI 受重力影响，会掉落、碰撞、滑动或翻倒。

稳定态

UI 落地后进入可用状态。
为了可玩性，平台类 UI 会有较高摩擦、较低弹性，避免乱飞。


规则 2：UI 有上下文限制

第一关里的 UI 不能无限带走。


规则 3：不同 UI 承担不同谜题职责

第一关强制区分 UI 用途：

文本：轻型平台 + 语义指令

窗口：重型平台 + 压机关

血条：代价资源 + 临时踏点

对话框：文本长度生成地形 + 遮挡

下拉菜单：结构展开 + 父子依附

复选框：状态切换 + 门禁逻辑


四、关卡完整流程


Stage 0：异常苏醒

教学目的

让玩家理解自己不是普通角色，而是系统里的异常光标实体。

场景布局

Caret-7 掉落在一片平坦区域。前方没有危险，只有一行闪烁教程：

Press [W] or [SPACE] to Jump

[SPACE] 一词偶尔故障抖动，鼠标悬停时会高亮。

玩家行为

玩家先尝试移动和跳跃。
系统确认玩家基本操作后，前方墙体亮起。

验证点

WASD / Space 基础移动

可交互 UI 高亮

鼠标悬停反馈

第四面墙第一次松动


Stage 1：跳不过去的墙

谜题 1：文字不是提示，是材料

场景布局

前方出现一堵高墙，正常跳跃无法通过。
上方仍然显示：

Press [W] or [SPACE] to Jump

其中 [SPACE] 可被拖拽。

解法

玩家用鼠标抓住 [SPACE]，将它从句子中拽下来。
松手后 [SPACE] 掉到地上，成为轻型平台。
Caret-7 踩着 [SPACE] 跳过高墙。

关键体验

玩家第一次意识到：
教程文字不是背景，而是物理资源。

防逃课设计

高墙后有一道“语义边界”。
如果玩家继续拖着 [SPACE] 往后走，它会逐渐乱码化：

Token lost semantic context.

随后 [SPACE] 变成无碰撞的 Broken Glyph，无法继续使用。

程序验证点

文本拆解

文本物理化

松手后重力掉落

上下文失效机制


Stage 2：任务窗口桥

谜题 2：菜单不是暂停层，是实体窗口

场景布局

玩家来到一个宽坑前。
背景墙上出现提示：

Press [TAB] to view objective.

按下 TAB 后，中央弹出 Quest Log：

Objective: Reach EXIT.

窗口可拖拽，有明显重量。

解法

玩家拖动 Quest Log 到坑中间。
松手后窗口坠落，卡在两个损坏数据柱之间，形成稳定桥面。
Caret-7 从窗口上跳过。

关键体验

系统菜单也是物理实体，而且比文本更重、更稳定。

加强版细节

坑中设置两个可落点：

平放窗口：标准解

斜靠窗口：可作为坡道冲过去

这样保留一点“涌现式解法”。

防逃课设计

过坑后进入“无 TAB 信号区”。
Quest Log 如果被拖离信号区，会逐渐透明并被吸回屏幕边缘。

系统提示：

Quest Log disconnected from signal range.

程序验证点

窗口类 UI 物理重量

拖拽滞后手感

UI 释放后稳定承载

区域信号绑定


Stage 3：疼痛的代价

谜题 3：血条不是生命，是可拆资源

场景布局

左上角首次出现 3 颗心形 HP。
前方依次是：

一小段地刺

一个绿色乱码毒水坑

毒水坑宽度需要 2 个心形 UI 才能稳定通过

此区域无法召唤 Quest Log。

系统提示：

HP integrity linked to physical layer.

玩家预期

玩家会本能避开地刺。
来到毒水前发现无法通过，也没有菜单可用。

标准解

玩家故意踩地刺。
每受伤一次，左上角一颗心形 UI 脱落，掉入场景。
玩家需要掉落两颗心，将它们拖入毒水坑作为踏点。

资源限制

每颗心进入毒水后会逐渐腐蚀。

腐蚀前足够玩家通过。

只剩 1 HP 时再受伤会死亡重置。

可选高难解

玩家也可以只掉一颗心，通过更精准跳跃过坑。
这作为隐藏技巧，不要求普通玩家掌握。

关键体验

掉血不是惩罚，而是把生命拆成了地形。

防逃课设计

心形离开伤害区域太远后会变成半透明残影，并回收到 HP 槽位。
这表示 HP 资源只能在伤害上下文中稳定实体化。

程序验证点

HP UI 与角色状态绑定

受伤生成物理 UI

UI 资源消耗

毒水腐蚀

代价交换机制


Stage 4：喋喋不休的 NPC

谜题 4：文本长度改变物理体积

场景布局

玩家来到一个高台断层下方。
断层旁站着损坏 NPC：Error_Bot。

高台上方有一束竖直扫描光线，玩家直接靠近会被弹回。

NPC 对话阶段

第一次点击：

Hi.

生成一个很小气泡，不能垫高。

第二次点击：

Still here?

生成中型气泡，可当小垫脚石，但不够到高台。

第三次点击：

Pathfinding failed. Dialogue buffer overflow. Please contact administrator...

生成一个巨大的长对话框。

标准解

玩家将巨型对话框拖到墙边，斜靠成坡。
同时对话框遮住扫描光线。
Caret-7 沿着对话框爬上高台。

关键体验

NPC 的废话越多，生成的物理面积越大。

防逃课设计

对话框离开 NPC 太远会出现拉伸线，随后断连并回收。
小气泡可以短距离移动，大气泡更重但范围更受限。

程序验证点

文本长度驱动 UI 尺寸

NPC 生成 UI

大型 UI 斜靠稳定性

UI 遮挡扫描机制

源对象依附


Stage 5：折叠菜单梯

谜题 5：UI 不只是物体，也可以展开成结构

这一段是第一关加入的结构类 UI 原型展示，用于预告第二章“界面层”。

场景布局

玩家爬上高台后，前方出现一面高墙。
墙边悬浮着一条损坏菜单栏：

FILE ▾

鼠标悬停时，FILE 高亮。

点击后展开：

FILE
New
Open
Save
Export
Exit

每个菜单项都有碰撞体。

标准解

玩家点击 FILE 展开菜单。
拖动父级菜单栏到墙边。
下拉菜单垂下，形成一串竖向阶梯。
玩家沿着菜单项跳上墙。

关键体验

下拉菜单不是一块砖，而是一种可以展开的结构。

与前面 UI 的区别

它的核心价值不是重量，也不是面积，而是：

展开 / 收起

父子依附

结构延展

碰撞体动态变化

防逃课设计

下拉菜单必须连接菜单栏信号才能保持展开。
离开本区域太远会自动折叠成一个短短的 FILE 块，不能继续当梯子。

程序验证点

展开 / 收起系统

父子 UI 依附

动态碰撞体

结构类 UI 原型


Stage 6：出口三重锁

谜题 6：不同 UI 必须分工合作

这一段是第一关的综合测试，同时加入状态类 UI 原型展示，用于预告第三章“逻辑层”。


场景布局

玩家终于看到绿色 EXIT 门，但门被三层锁住。

门上显示：

EXIT LOCKED
Missing: WEIGHT / PERMISSION / COMMAND

场景中有：

一个红色重量按钮

一个短沟壑

一个授权面板

空中悬浮文字 LEVEL CLEAR

当前区域可召唤的 Quest Log

授权面板显示：

Exit Permission
[ ] I am not a bug
[ ] Enable safe exit
[ ] Accept deletion


第一层：WEIGHT

解法

玩家按 TAB 召唤 Quest Log。
把 Quest Log 拖到红色按钮上。
窗口落下后压住按钮。

门状态更新：

WEIGHT confirmed.

设计目的

验证窗口是重型 UI，适合压机关。
同时形成一物多用限制：Quest Log 必须留在按钮上，不能拿去搭桥。


第二层：PERMISSION

玩家预期

玩家可能先勾选：

I am not a bug

系统报错：

Identity mismatch.

因为 Caret-7 的确是异常变量。

标准解

玩家勾选：

Enable safe exit

门状态更新：

PERMISSION confirmed.

设计目的

展示复选框不是平台，而是状态 UI。
它改变的是系统逻辑。


隐藏变体解：错误选项生成资源

如果玩家勾选：

Accept deletion

系统弹出倒计时：

Deleting in 5...4...3...

随后出现确认框：

Are you sure?
YES / NO

玩家可以把 NO 按钮拖下来作为临时平台，或者压住机关。
这个作为隐藏解或彩蛋，不作为标准必解。

设计目的

展示“错误选择也可能变成资源”，延续负反馈转资源的核心哲学。


第三层：COMMAND

门上显示：

Missing command: CLEAR

空中悬浮：

LEVEL CLEAR

解法

玩家把 CLEAR 从文字中拆下来。
但这一次不是拿来铺路，而是拖进 EXIT 门的命令槽。

门状态更新：

COMMAND confirmed.

设计目的

展示文本不只是物理平台，也有语义属性。


最后过沟

门已经打开，但前方还有一道短沟。
玩家可以把剩下的 LEVEL 拆成轻型文本块，铺在沟上通过。

注意：
Quest Log 不能拿走，因为拿走后按钮弹起，门会重新锁上。

关键体验

不同 UI 不能互相替代，必须按规则分工。


五、最终出口反馈

当玩家进入 EXIT，系统显示：

Level Clear.
External pointer interference confirmed.
Caret-7 classification updated: BUG / SENTIENT.

随后画面短暂黑屏，进入章节选择或第二关预告。

远处闪现下一章提示：

Interface Layer unlocked.
Dropdown structures detected.
